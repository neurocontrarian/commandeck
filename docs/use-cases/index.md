# Use Cases

Real-world walkthroughs showing how people use Commandeck day to day. Pick the one closest to you:

- **[Linux Beginner Guide](beginner.md)** — new to Commandeck (or to Linux)? Start here. Works fully on the free tier, no SSH needed.
- **[Home Server Management](home-server.md)** — one-click maintenance for a Plex / Pi-hole / NAS box over SSH. *(Pro)*
- **[Development Workflow](dev-workflow.md)** — a developer's command palette: build, test, deploy and check logs, locally and on a remote VPS. *(remote parts are Pro)*
- **[Manage a Homelab Fleet](homelab.md)** — drive several machines from one grid with multi-machine buttons and execution profiles. *(Pro)*
- **[Drive Commandeck from a Local AI](local-ai.md)** — let a local or cloud AI build and run your buttons via MCP. *(Pro)*

!!! note
    Guides marked *(Pro)* rely on SSH machines, multi-machine buttons, execution profiles, or AI/MCP — all [Commandeck Pro](../pro.md) features. The free tier covers local use (start with the Beginner Guide).
