# Cas d'utilisation

Des guides concrets montrant comment on utilise Commandeck au quotidien. Choisissez celui qui vous ressemble :

- **[Guide du débutant Linux](beginner.fr.md)** — nouveau sur Commandeck (ou sous Linux) ? Commencez ici. Fonctionne entièrement dans la version gratuite, sans SSH.
- **[Gestion d'un serveur domestique](home-server.fr.md)** — maintenance en un clic d'une machine Plex / Pi-hole / NAS via SSH. *(Pro)*
- **[Workflow de développement](dev-workflow.fr.md)** — une palette de commandes pour développeur : build, test, déploiement et consultation des logs, en local et sur un VPS distant. *(les parties distantes sont Pro)*
- **[Gérer un parc homelab](homelab.fr.md)** — pilotez plusieurs machines depuis une seule grille avec des boutons multi-machines et des profils d'exécution. *(Pro)*
- **[Piloter Commandeck depuis une IA locale](local-ai.fr.md)** — laissez une IA locale ou cloud créer et exécuter vos boutons via MCP. *(Pro)*

!!! note
    Les guides marqués *(Pro)* reposent sur les machines SSH, les boutons multi-machines, les profils d'exécution ou l'IA/MCP — toutes des fonctionnalités [Commandeck Pro](../pro.fr.md). La version gratuite couvre l'usage local (commencez par le Guide du débutant).
